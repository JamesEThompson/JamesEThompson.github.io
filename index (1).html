// 01 — Project Description

What This Project Does

This project simulates the real-time control systems found inside a modern fighter jet — specifically inspired by avionics and weapon management systems used in programs like Northrop Grumman's aerospace work in Orlando, Florida. It runs on an ESP32 microcontroller using FreeRTOS, the same real-time operating system used in safety-critical embedded systems across aerospace and defense.

The system has two weapon controls: a missile button that fires a single weapon (one LED flash), and a machine gun button that simulates a cannon burst (12 rapid LED flashes). A potentiometer acts as an altimeter — turn it below 25% and a red LED activates alongside repeated altitude warnings in the serial monitor, just like a real aircraft's ground proximity warning system.

Each feature runs as its own FreeRTOS task with a defined priority and deadline. Weapon response tasks are Hard real-time — a missed deadline means a failed weapon system. The altitude monitor is Soft real-time — a slight delay is annoying but not catastrophic. This mirrors how real aerospace engineers classify requirements.

🎯 Missile Fire

Press the green button and the weapon LED lights for 300 ms — one missile away. A binary semaphore ensures the button task and LED task never conflict.

🔫 Machine Gun Burst

Press the yellow button for 12 rapid LED flashes simulating cannon rounds. Execution time varies with burst length — a true variable-time FreeRTOS task.

⚠️ Altitude Warning

Turn the potentiometer low and the red LED activates. The serial monitor floods with ALTITUDE warnings — just like a GPWS in a real cockpit.

🔧 Built With FreeRTOS

Four concurrent tasks run simultaneously on the ESP32, each with defined priorities so critical systems always preempt lower-priority ones.
