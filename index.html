// 01 — Project Description

What This Project Does

This project simulates the real-time control systems found inside a modern fighter jet — specifically inspired by the avionics and weapon management systems used in programs like Northrop Grumman's aerospace work in Orlando, Florida. It runs on an ESP32 microcontroller using FreeRTOS, the same type of real-time operating system used in safety-critical embedded systems across aerospace and defense.

The system has two weapon controls: a missile button that fires a single weapon (one LED flash), and a machine gun button that simulates a cannon burst (36 rapid LED flashes). A potentiometer acts as an altimeter — turn it below 25% and a red LED activates alongside repeated altitude warnings in the serial monitor, just like a real aircraft's ground proximity warning system.

A dedicated emergency eject button is wired directly to a hardware interrupt on GPIO 23. When pressed, it bypasses the normal task scheduler entirely and fires an ISR instantly — the highest priority response in the entire system, simulating a pilot ejection sequence. A separate heartbeat LED on GPIO 26 blinks every 500 ms to prove the system is meeting its hard deadlines in real time. Attach the Wokwi Logic Analyzer to GPIO 26 to verify the timing on an oscilloscope-style display.

Each feature runs as its own FreeRTOS task with a defined priority and deadline. The weapon and eject tasks are Hard real-time — a missed deadline means a failed weapon system or a failed eject sequence. The altitude monitor and heartbeat are Soft real-time — a slight delay is undesirable but not catastrophic. This mirrors how real aerospace engineers classify requirements.

🎯 Missile Fire

Press the green button and the weapon LED lights for 300 ms — one missile away. A binary semaphore ensures the button task and LED task never conflict. Can also be triggered remotely from the WiFi web page.

🔫 Machine Gun Burst

Press the yellow button to trigger 36 rapid LED flashes simulating cannon rounds. Execution time varies with burst count — a true variable-time FreeRTOS task.

⚠️ Altitude Warning

Turn the potentiometer low and the red LED activates. The serial monitor floods with timestamped ALTITUDE warnings — just like a GPWS in a real cockpit. Altitude data is also passed to the weapon task via an internal queue.

🚨 Emergency Eject ISR

The red eject button on GPIO 23 fires a hardware interrupt — the fastest possible response in the system. It bypasses all task scheduling and immediately triggers the eject sequence, flashing both LEDs 10 times to signal ejection.

💙 Heartbeat Monitor

A dedicated blue LED on GPIO 26 toggles every 500 ms. This is the determinism proof — if the heartbeat stays consistent, all hard deadlines are being met. Attach the Wokwi Logic Analyzer to GPIO 26 for full credit verification.

🔧 Built With FreeRTOS

Six concurrent tasks plus a hardware ISR run simultaneously on the ESP32, each with defined priorities so critical systems always preempt lower-priority ones. WiFi connects to Wokwi-GUEST and serves a live control page.

Tasks, Priorities & Deadlines

Task Priority Period Type Scenario

EjectTask 4 Aperiodic HARD Responds to ejectSem from ISR — flashes both LEDs 10x to simulate pilot ejection sequence
MissileBtn 3 10 ms poll HARD Polls GPIO 18 with 50 ms debounce — posts to missileSem on press
GunBtn 3 10 ms poll HARD Polls GPIO 19 with 50 ms debounce — posts to gunSem on press
